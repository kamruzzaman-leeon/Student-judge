Student Judge (update 0.1)

Student Judge System is a web application that can used for schools, colleges, coaching centers or any kind of educational institutions. It is such a platform where students can continue their studies and skill developments from anyplace to examine themselves at their own will. It is a platform for a certain group of students can brush up their knowledge by watching subject wise video lecture, blogs and attending the online based exams. They can also track their progress with others.

Functional Requirement:
1. User Authentication: There will be two types of users. They are:
1.1. Admin: Admin will control all the managements of the software.
1.2. Teachers: Teachers can upload videos, write blogs, make questions, and check script.
1.3. Student: Students get all the required services like videos, blogs, e-Books, and so on along with participate in exams. 
2.    User Profile: Everyone has the same things in profile.
2.1. Edit information: User can edit his profile information anytime they wants.
2.2. Delete account: User can delete their account whenever they wants.
2.3. Logout: User can log out of their account after completing their work.
3. Course level: Course level is related to its category. The category divided the course level.
	Category:
	Academic (subcategory class: 1-10).
	Skill development.
4. Examination:
4.1. MCQ: There will be quizzes to brush up their selves in a very short time. MCQ checked automatically (write and upload answer into the server). If anyone answers wrong then suggest the topic for improvement.
4.2. Written: There will be a written exam which is checked manually. But copy/paste will be checked by software.
5. Progress report:
5.1. Self-report: One can see one's progress report.
5.2. Other-report: A student can see other marks as well. So that he can observe his position.
6. Mail service: This service provides a student course-related information and progress report and new blog arrival news.


Technologies
Frontend: HTML, CSS, BOOTSTRAP, JAVASCRIPT, AJAX
Backend: Django
Database: MySQL
